About the app

This web service is built using Python flask. For data persistence, I used SQLAlchemy toolkit that also provides Object Relational Mapper. For persistence layer, SQLAlchemy is configured to use sqlite3 which is hosted locally on the desk. Since we are dealing with APIs that could accept and return different types of data

I wanted to experiment with data serialization tool which is called Marshmallow. I am using those two tools for the first time; however, it is very interesting approach to interact with database and prepare API responses.

With the help of Marshmallow Schemas and Swagger I was able to auto generate API docs with little markups. There were some limitations in this approach such as when I wanted to auto document an endpoint that accepts GET 'arguments'. Further more, the app has been containerized and deployed on a remote Heroku machine, It should be running now :)

Features List

  • Reset data in the app any time, before running, or even when the app is running

  • App can run on any machine that uses Docker or Docker Toolbox(windows)

  • The app is already live, so you do not need to run

  • On this application go to API Docs to see full list of endpoints

    • Features for the challenge

    Products
    • Get a list of all products in the system
    • Filter product by `min_price` `max_price` `available` `page` `limit`
    • Get specific product using its id
    • Products have `visits` fields that increments every time the product has been requested by id
    Purchase
    • Purchase any product with any quantity you want: simple POST - for now
    Cart
    • Load all the products in the cart with all requested quantities
    • Cart will also tell you what is the inventory of each product placed in the cart
    • Delete a product from the cart
    • Add or Update a product in the cart
    • Checkout cart to process all the products; however, it will also return a cart with the products that you could not purchase because their inventory is empty. They will be kept in the inventory.
    Reset
    • Reset the data in the database to its previous state, in case things got wrong :(

How to build and run

I have based my work on Python 3.5

  • Using Virtual Environment

    pip install virtualenv
    // Linux 
    virtualenv venv
    source bin/activate
    // windows 
    python -m virtualenv venv
    venv\Script\Activate
    // installing the required libs 
    pip install -r requirements
    //setup the database 
    python setup.py
    run the app 
    python main.py
    // the app should start on http://localhost:5000

  • Using docker

    docker build -t dockershopifyapp .
    // run the docker container 
    docker run -p 5000:5000 dockershopifyapp
    // the app should start on http://localhost:5000

  • Using DockerToolbox (windows)

    docker-compose up
    // the up should start on http://192.168.99.100:5000
